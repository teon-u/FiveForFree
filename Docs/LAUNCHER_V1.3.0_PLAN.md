**작성자:** 분석팀장

## 1. 역할 분석

### 1.1 런처 vs 대시보드 역할 구분

| 구분 | 런처 (launcher.py) | 대시보드 (Dashboard.jsx) |
|------|-------------------|------------------------|
| 목적 | 로컬 실행 환경 관리 | 실시간 모니터링/분석 |
| 대상 | 개발자/운영자 | 트레이더/분석가 |
| 환경 | 데스크톱 (Tkinter) | 웹 브라우저 (React) |
| 접근 | 로컬 전용 | 원격 가능 |

### 1.2 현재 기능 비교

**런처 (현재 기능):**
- 프로세스 관리 (라이브/페이퍼 트레이딩, API, 프론트엔드, 백테스트)
- 빠른 작업 (결과 보기, 대시보드, 로그, 설정, 데이터 확인, 티커 목록)
- 로그 출력 뷰어
- 단축키

**대시보드 (현재 기능):**
- 실시간 티커 가격/예측 표시 (TickerGrid)
- 차트 및 모델 비교 (ChartModal, ModelDetailModal)
- 필터 및 검색 (FilterBar)
- 워치리스트 관리 (WatchlistPanel)
- 발견 기능 (DiscoveryBanner/Panel)
- 알림 센터 (NotificationCenter)
- 내보내기 (ExportModal)

## 2. 제안 기능 (대시보드와 겹치지 않음)

### 2.1 모델 학습 관리 ⭐ (최우선)

**목적:** 런처에서 모델 학습 시작/중지/모니터링

**기능:**
```
[모델 학습 관리]
- 학습 시작: 선택한 티커의 모델 학습 시작
- 학습 중지: 실행 중인 학습 종료
- 진행률 표시: 에폭, 손실값, 예상 남은 시간
- 학습 로그: 실시간 학습 로그 표시
```

**UI 목업:**
```
+------------------------------------------+
| 모델 학습 관리                        [X] |
+------------------------------------------+
| 티커 선택: [AAPL v]  모델: [Ensemble v]  |
|                                          |
| [학습 시작]  [중지]  [결과 보기]         |
|                                          |
| 진행률: [████████░░░░░░░░] 52%           |
| 에폭: 52/100 | 손실: 0.0234 | ETA: 5분   |
|                                          |
| [학습 로그]                              |
| Epoch 52: loss=0.0234, val_loss=0.0256   |
| Epoch 51: loss=0.0241, val_loss=0.0263   |
| ...                                       |
+------------------------------------------+
```

**단축키:** `Ctrl+M` (Model)

### 2.2 데이터 수집 제어

**목적:** 데이터 수집 시작/중지/스케줄링

**기능:**
```
[데이터 수집 제어]
- 즉시 수집: 선택한 티커 데이터 즉시 수집
- 수집 중지: 실행 중인 수집 중단
- 스케줄링: 수집 시간 설정 (매일 XX시)
- 수집 상태: 현재 수집 중인 티커 표시
```

**UI 목업:**
```
+------------------------------------------+
| 데이터 수집 제어                      [X] |
+------------------------------------------+
| [즉시 수집] [중지] [스케줄 설정]         |
|                                          |
| 현재 상태: 수집 중 (3/50 티커)           |
| 진행률: [████░░░░░░░░░░░░] 6%            |
|                                          |
| [최근 수집]                              |
| AAPL: 1,234 bars (2분 전)                |
| MSFT: 1,198 bars (3분 전)                |
| GOOGL: 수집 중...                        |
+------------------------------------------+
```

**단축키:** `Ctrl+D` (Data)

**목적:** CPU, RAM, GPU 사용량 실시간 표시

```
- 디스크 사용량
- 프로세스별 리소스 사용량
```

**UI 목업:**
```
+------------------------------------------+
| 시스템 모니터                         [X] |

| CPU: [██████░░░░] 62%                    |

| Disk: [██████████] 234 GB 사용 가능       |

| [프로세스별]                             |

+------------------------------------------+
```

### 2.4 환경 설정 마법사

**목적:** 초기 설정 및 종속성 검증

**기능:**
```
[환경 설정 마법사]
1. Python 버전 확인
2. 필수 패키지 설치 확인
3. .env 파일 설정
4. 데이터베이스 초기화
5. API 키 유효성 검증
6. 모델 디렉토리 확인
```

**UI 목업:**
```
+------------------------------------------+
| 환경 설정 마법사                      [X] |
+------------------------------------------+
| [1/6] Python 버전 확인                   |
| ✓ Python 3.11.5 설치됨                   |
|                                          |
| [2/6] 필수 패키지                        |
| ✓ torch 2.1.0                            |
| ✓ fastapi 0.104.1                        |
| ✗ xgboost 미설치 → [설치]                |
|                                          |
| [3/6] .env 설정                          |
| ⚠ POLYGON_API_KEY 미설정 → [설정]        |
|                                          |
|        [이전] [다음] [건너뛰기]          |
+------------------------------------------+
```

**단축키:** `Ctrl+W` (Wizard)

### 2.5 향상된 로그 뷰어

**목적:** 로그 검색, 필터링, 내보내기

**기능:**
```
[향상된 로그 뷰어]
- 로그 레벨 필터 (DEBUG, INFO, WARNING, ERROR)
- 키워드 검색
- 시간 범위 필터
- 로그 내보내기 (TXT, CSV)
- 자동 스크롤 토글
- 로그 하이라이팅
```

**UI 목업:**
```
+----------------------------------------------------------+
| 로그 뷰어                                             [X] |
+----------------------------------------------------------+
| 검색: [___________] [레벨: ERROR v] [시간: 최근 1시간 v] |
| [새로고침] [내보내기] [자동스크롤: ON]                    |
+----------------------------------------------------------+
| 2025-12-26 22:45:32 [ERROR] API 연결 실패: timeout       |
| 2025-12-26 22:45:30 [WARNING] 메모리 사용량 높음: 85%    |
| 2025-12-26 22:45:28 [INFO] 예측 완료: AAPL               |
| ...                                                       |
+----------------------------------------------------------+
| 표시: 156개 / 전체: 12,345개                              |
+----------------------------------------------------------+
```

**단축키:** `Ctrl+L` (Log)

### 2.6 프로세스 자동 재시작

**목적:** 프로세스 충돌 시 자동 복구

**기능:**
```
[자동 재시작 설정]
- 프로세스별 자동 재시작 ON/OFF
- 재시작 횟수 제한 (예: 3회)
- 재시작 간격 (예: 10초)
- 재시작 알림 (토스트 메시지)
```

**UI 위치:** 각 프로세스 버튼 옆 체크박스

---

### 2.7 백업 및 복원

**목적:** 설정, 모델, 데이터 백업/복원

**기능:**
```
[백업 관리]
- 설정 백업 (.env, config.py)
- 모델 백업 (models/ 디렉토리)
- 데이터베이스 백업 (stocks.db)
- 자동 백업 스케줄 (매일/매주)
- 복원 마법사
```

**UI 목업:**
```
+------------------------------------------+
| 백업 관리                             [X] |
+------------------------------------------+
| [설정 백업] [모델 백업] [DB 백업] [전체] |
|                                          |
| 최근 백업:                               |
| 2025-12-26 22:00 - 전체 (245MB)          |
| 2025-12-25 22:00 - 전체 (243MB)          |
|                                          |
| 자동 백업: [매일 v] [22:00 v] [ON]       |
|                                          |
|        [복원] [삭제] [설정]              |
+------------------------------------------+
```

**단축키:** `Ctrl+B` (Backup)

## 3. 사용성 개선 사항

### 3.1 UI/UX 개선

| # | 항목 | 현재 | 개선안 |
|---|------|------|--------|
| 1 | 상태 표시 | 텍스트만 | 컬러 인디케이터 (🟢실행/🔴중지/🟡로딩) |
| 2 | 버튼 배치 | 수평 나열 | 그룹화된 카드 레이아웃 |
| 3 | 단축키 | 툴팁에만 | 버튼에 직접 표시 (예: "결과 (Ctrl+O)") |
| 4 | 다크 모드 | 없음 | 라이트/다크 테마 전환 |
| 5 | 폰트 크기 | 고정 | 설정 가능 (소/중/대) |
| 6 | 창 크기 | 고정 | 리사이즈 가능, 레이아웃 저장 |

### 3.2 사용자 경험 개선

| # | 항목 | 현재 | 개선안 |

| # | 항목 | 현재 | 개선안 |
|---|------|------|--------|
| 1 | 키보드 네비게이션 | 일부 | Tab 순서 개선, 포커스 표시 |
| 2 | 고대비 모드 | 없음 | 고대비 테마 추가 |
