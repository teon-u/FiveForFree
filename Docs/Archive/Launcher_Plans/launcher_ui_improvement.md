## 1. 현재 UI 분석

### 1.1 파일 정보
- **파일 위치**: `projects/FiveForFree/launcher.py`
- **총 코드 라인**: 약 2,500줄
- **GUI 프레임워크**: Tkinter (ttk 스타일 사용)
- **창 크기**: 900x850 (최소 800x750)

### 1.2 현재 구조
```
+------------------------------------------+
|  메뉴바 (파일 | 도움말)                    |
+------------------------------------------+
|  컨트롤 패널                              |
|  ┌─ 실시간 트레이딩: [상태] [시작] [정지]  |
|  ├─ 모의 트레이딩: [상태] [시작] [정지]    |
|  ├─ API 서버: [상태] [시작] [정지] [API문서]|
|  ├─ 프론트엔드: [상태] [시작] [정지]       |
|  ├─ 백테스트: [기본실행] [고급실행]        |
|  └─ 빠른 작업: [12개 버튼 한 줄에 나열]    |
+------------------------------------------+
|  로그 출력 (ScrolledText)                 |
|  [로그 지우기]                            |
+------------------------------------------+
|  상태바                                   |
+------------------------------------------+
```

## 2. 발견된 사용성 문제점

### 2.1 빠른 작업 영역 과밀화 (심각도: 높음)
**현상**: 12개의 버튼이 한 줄에 배치되어 시각적 혼잡함 유발
- 결과 보기, 대시보드, 로그 보기, 설정 편집, 데이터 확인, 티커 목록, 모델 학습, 시스템, 데이터 수집, 백업, 로그, 설정

**문제점**:
- 버튼 식별이 어려움
- 기능 간 연관성 파악 불가
- 화면이 좁으면 버튼이 잘림

### 2.2 일관성 없는 버튼 크기 (심각도: 중간)
**현상**: width=6, 8, 10 등 다양한 크기 혼재
```python
# 현재 코드
ttk.Button(..., width=10)  # 결과 보기
ttk.Button(..., width=8)   # 시스템
ttk.Button(..., width=6)   # 로그, 설정
```

**문제점**:
- 시각적 통일성 부족
- 버튼 중요도가 명확하지 않음

### 2.3 메뉴 구조 부족 (심각도: 중간)
**현상**: 파일, 도움말 메뉴만 존재
- 많은 기능이 빠른 작업 버튼으로만 접근 가능
- 기능 발견성(Discoverability) 저하

### 2.4 논리적 그룹핑 부재 (심각도: 높음)
**현상**: 빠른 작업 버튼들이 논리적으로 분류되지 않음
- 데이터 관련: 데이터 확인, 티커 목록, 데이터 수집
- 시스템 관련: 시스템, 백업
- 분석 관련: 결과 보기, 로그 뷰어
- 설정 관련: 설정 편집, 설정 마법사

### 2.5 상태 표시 가시성 부족 (심각도: 중간)
**현상**: 프로세스 상태가 작은 라벨로만 표시
- "중지됨" (빨강), "실행중" (녹색) 텍스트만 사용
- 한눈에 전체 상태 파악이 어려움

### 2.6 단축키 접근성 (심각도: 낮음)
**현상**: 단축키 정보가 툴팁에만 존재
- 키보드 사용자가 단축키를 발견하기 어려움
- 메뉴나 버튼에 단축키 표시 없음

## 3. 개선안

### 3.1 탭 기반 레이아웃 도입 (권장)
```
+------------------------------------------+
|  [트레이딩] [분석] [데이터] [시스템]       |
+------------------------------------------+
|  탭 별 관련 기능 그룹                      |
+------------------------------------------+
```

**트레이딩 탭**: 실시간/모의 트레이딩, 백테스트
**분석 탭**: 결과 보기, 대시보드, 로그 뷰어
**데이터 탭**: 데이터 확인, 티커 목록, 데이터 수집
**시스템 탭**: 시스템 모니터, 백업, 설정

### 3.2 메뉴 구조 확장
```
파일
  ├─ 프로젝트 폴더 열기
  ├─ 결과 폴더 열기
  ├─ 로그 폴더 열기
  └─ 종료

도구
  ├─ 데이터 수집
  ├─ 모델 학습
  ├─ 백테스트 실행 ▶
  │   ├─ 기본 실행
  │   └─ 고급 실행
  └─ 시스템 모니터

설정
  ├─ 설정 편집
  ├─ 환경 설정 마법사
  └─ 백업/복원

도움말
  ├─ 단축키 목록
  ├─ 사용자 가이드
  └─ 정보
```

### 3.3 사이드바 + 메인 콘텐츠 레이아웃
```
+--------+--------------------------------+
| 사이드  |  메인 콘텐츠                    |
| 바     |  (선택된 기능 표시)              |
|        |                                |
|  ▶실시간|                                |
|  ▶모의  |                                |
|  ▶API  |                                |
|  ▶웹   |                                |
|  ------+                                |
|  도구  |                                |
|  설정  |                                |
+--------+--------------------------------+
```

### 3.4 상태 대시보드 강화
```
+------------------------------------------+
|  시스템 상태                               |
|  ┌────────┬────────┬────────┬────────┐   |
|  │ 실시간  │ 모의   │  API   │  웹    │   |
|  │  ⚫     │  ⚫    │  🟢    │  🟢   │   |
|  │ 중지    │ 중지   │ 실행중  │ 실행중 │   |
|  └────────┴────────┴────────┴────────┘   |
+------------------------------------------+
```

### 3.5 버튼 표준화
- 모든 버튼 width=10 통일
- 아이콘 + 텍스트 조합 사용 (선택적)
- 단축키 표시: `[시작 (F5)]`

### 3.6 반응형 레이아웃
- 창 크기에 따른 버튼 재배치
- Grid 또는 Flow 레이아웃 적용
- 스크롤바 추가로 모든 버튼 접근 가능

## 5. 빠른 개선안 (즉시 적용 가능)

### 5.1 빠른 작업 버튼 2행 분리
```python
# 현재: 12개 버튼 한 줄
# 개선: 6개씩 2행

# 1행: 분석/보기
quick_row1 = ttk.Frame(parent)
ttk.Label(quick_row1, text="분석/보기:", width=12)
# 결과 보기, 대시보드, 로그 보기, 로그 뷰어

# 2행: 관리/설정
quick_row2 = ttk.Frame(parent)
ttk.Label(quick_row2, text="관리/설정:", width=12)
# 데이터 확인, 티커 목록, 모델 학습, 데이터 수집, 시스템, 백업, 설정
```

### 5.2 구분선 추가
```python
ttk.Separator(parent, orient=tk.HORIZONTAL).pack(fill=tk.X, pady=5)
```

### 5.3 버튼 폭 통일
```python
BUTTON_WIDTH = 10  # 상수로 정의
ttk.Button(..., width=BUTTON_WIDTH)
```
